Introduction

Good day. If you're here, you're probably interested in learning a little more about me. You are in the right place. Please excuse the ugly mug shot, I'm not exactly photogenic and I took this one in a hurry. It's a slight improvement on the one in my Linkedin and Github account.

Below are a few details, mostly about my IT background. At the very bottom, there is a bit more information about my other interests. It is for these reasons that software development and I fit together well.

Feel free to skip past anything that doesn't interest you, I've tried to put something in here for everyone which means things can get a little long winded in places.

Other Related Skills

Other technologies related to sooftware development that I have skills in include GIT CLI (Bash Git) which I have began using recently in place of GitHub Desktop of which I am also familiar. Naturally, I have a GitHub account that I use alongside these. For development environments, I use Visual Studio 2022 (Community Edition) for compiled code. For Interpreted (web) based code I use Visual Studio Code, having moved on from Notepad++ which was my main-stay for quite a while.

I use Unity Hub for my Unity 3D work which integrates well with Visual Studio. It's quite nice for a visual environment and i found it quite easy to get the hang of. I use Audacity if I need to edit audio files, though that's not often. I wish to learn Blender for modelling etc, but it's a lower priority right now.

For web based work, I run APACHE server on my PC through XAMPP. I used to have an old PC that I installed Linux onto and ran within a local network as a substitute for this. I found this better for testing across different environments and different browsers. I'd like to do that again given the resources. Whilst on the subject of the web, I dabbled in the past with 3rd party CMS systems including Joomla and Wordpress. Whilst I prefer to hand code sites, I did find Joomla quite customisable and not too tricky to build templates for.

For image editing, my main choice is GIMP which being free, makes it all the better. I'm not an expert user, but I can get done what I need to. For lo-fo / mid-fi prototype interfaces, my choice of tool is MS Powerpoint. For practically anything else, I find Google is my friend and there are endless resources out there. That said, I HATE copying and pasting someone else's code. I am more prone to learning from an example and making my own version, given the choice.

Environments (O/S)

Whilst a majority of my time has been spent within the MS Windows environment, I really enjoyed the days of the MS DOS command prompt. I have played quite a bit with Linux in the past, I even had a stint with Unix and an old Xenix server that ran a number of Wyse terminals. I helped with routine maintenance on this system.

I owned a Macbook for a few years which I loved and got used to the OS-X environment. Having the ability to open up a terminal and get to a 'Bash' style environment was appealing to me too.
